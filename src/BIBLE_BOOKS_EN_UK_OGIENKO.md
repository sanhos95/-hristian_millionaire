## Часті варіанти в англійських посиланнях (нормалізувати до канону)

| Як може бути в тексті | Нормалізувати до |
|------------------------|------------------|
| `Psalm` (однина) | Psalms |
| `I Samuel`, `II Kings`, `III John` (римські) | 1 Samuel, 2 Kings, 3 John |
| `Song of Songs` | Song of Solomon |
| `Revelations` (помилкова множина) | Revelation |
| `Exodous` (опечатка в `questions.json`) | Exodus |
| `Dan` (скорочення) | Daniel |
| `Philemon` | Philemon (як книга; збігається з каноном) |

## Примітки для наступного кроку (новий JSON з Огієнка)

- **Категорії** з `questions.json` (`categories`) переносити без зміни — вони знадобляться для фільтрації/тем.
- **Питання й відповіді** перекладати/формулювати українською; **цитату з тексту** брати з `verses.json` (Огієнко) за `book_number` + `chapter` + `verse`.
- **`reference`** залишати англійським як джерело або дублювати українським виглядом (`reference_uk`), як у `questions-ogienko.json`.

## Поле `wrongAnswers` у `questions-uk-ogienko.json` / `questions-ogienko.json`

- Масив з **трьох** рядків — невірні варіанти для MCQ (разом із `answer` буде 4 варіанти).
- Зміст: **осмислені правдоподібні** дистрактори (важко відрізнити від правильної відповіді за змістом питання), а не випадкові слова з вірша. Канонічний список у **`src/wrongAnswersPlausible.json`** (генерується з `scripts/gen-wrongAnswersPlausible.mjs`).
- Застосувати до питань: `npm run apply:wrong-answers`. Після правок масиву в `gen-wrongAnswersPlausible.mjs`: `npm run gen:wrong-answers && npm run apply:wrong-answers`.
