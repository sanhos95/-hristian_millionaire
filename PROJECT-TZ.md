# Біблійний квіз «Мільйонер» — план і технічне завдання (v0)

Документ для людини та для AI: мета проєкту, межі версії, стек, **фактична поведінка коду**, структура файлів, критерії готовності. **Після кожної зміни логіки або UI — оновлювати цей файл**, щоб наступний крок (людина чи AI) швидко орієнтувався.

---

## 1. Мета продукту

Веб-сайт з квізом у стилі класичного «Хто хоче стати мільйонером»: сходинки, чотири варіанти відповіді, обмежений набір підказок. Тематика — Біблія (питання та відповіді з канонічного тексту / загальноприйнятих фактів про книги, персонажів, події).

---

## 2. Обсяг поточної реалізації (v0 у коді)

**Є зараз:**

- Одна сторінка без бекенду: Vue 3 + Vite + TypeScript + Tailwind v4, дані на клієнті.
- **Банк питань:** основний контент — **`src/questions-uk-ogienko.json`** (українські тексти, `categories` з джерела, поля **`gameTopic`** для меню теми та **`ladderStep`** 1…10 — рівномірні децилі за внутрішньою оцінкою «жорсткості» у `annotate-ogienko-metadata.mjs`); збірка в TypeScript: **`src/data/ogienkoQuestions.gen.ts`** (`npm run refresh:ogienko-bank`). У грі: `src/data/questions.ts` об’єднує `OGIENKO_QUESTIONS` + `extraQuestions.ts`. У кожній грі рівно **10** питань (`DECK_SIZE`): у колоді **раунди 1…10** відповідають сходинкам **1…10** (спочатку легші, до кінця важчі); за потреби — дублікат id з суфіксом `__dN`.
- **Перед грою:** лише вибір **теми** (`TOPIC_MENU` у `topics.ts`). **Таймер на кожне питання** — **`TIMER_SECONDS_PER_QUESTION`** у `topics.ts` (**120 с**). Якщо час вийшов — **програш раунду** (`timedOut`), екран результату з поясненням «Час вичерпано»; таймер очищується при відповіді та при `onScopeDispose`.
- Драбина балів **динамічна** під довжину колоди (`prizeLadderForCount` у `questions.ts`, до 15 сходинок у таблиці + розширення при потребі).
- Екрани: **меню (тема)** → **гра** → **результат** («Грати знову» скидає до вибору теми).
- Підказки:
  - **50:50** — ховає два невірні варіанти; **лише один раз за гру** (прапорець `fiftyUsed` не скидається в `loadRound`).
  - **Запитати ШІ** — стовпчики % по A–D; **правильний варіант завжди з найбільшим відсотком**; **лише один раз за всю гру** (до нової сесії «Почати гру»).
  - **Вірш** — модальне вікно з хрестиком; **лише один раз за всю гру**; показує `verseLocation` + `verseText`, а якщо вірша немає — текст: «Упс... до даного питання вірш відсутній».
- Після вибору відповіді підказки **блокуються** до наступного питання.
- **Без** таймера, **без** «безпечних» сходинок: при помилці гра закінчується, бали = останній **повністю** пройдений рівень (див. `earnedPoints` у `useGame.ts`).

**Поза межами (ідеї на потім):**

- Таймер, звуки, безпечні сходинки, бекенд, облік гравців, редактор питань, інші мови.

## 5. Модель даних і стану

**Питання (`BibleQuestion` у `bibleQuestionType.ts`, банк у `questions.ts`):**

- `id`, `text`, `answers`, `correctIndex`, обов’язково **`topic`** (`QuestionTopic` — див. `topics.ts`) та **`ladderStep`** (`1`…`10` — місце на «драбині» складності в банку).
- опційно `reference`, `correctAnswerWords`.

**Сирі теги в JSON (`categories` у `questions-uk-ogienko.json`):** `old_testament`, `new_testament`, `gospels`, `miracles`, `judges`, `kings_judges`, `exodus`, `prophets`, `psalms`, `psalms_`, `proverbs`, `letters_of_paul`, `letter_of_paul` (останній нормалізується до `letters_of_paul`).

**Мапінг `categories` → `gameTopic` / поле `topic` у грі** (скрипт `scripts/annotate-ogienko-metadata.mjs`, пріоритет згори вниз): явні питання про місто за формулюванням питання → `cities`; інакше `exodus` → `manna_desert`; `psalms` \| `psalms_` \| `proverbs` → `books`; `letters_of_paul` → `letters_paul`; `gospels` → `gospels`; `judges` \| `kings_judges` → `judges_kings`; `miracles` → `miracles`; `prophets` → `prophets`; `new_testament` → `nt_general`; `old_testament` → `ot_general`; інакше `general`.

**Розподіл за ігровою темою (205 питань Огієнка):** `judges_kings` 29, `gospels` 43, `miracles` 18, `ot_general` 45, `manna_desert` 27, `nt_general` 23, `cities` 7, `prophets` 7, `books` 3, `letters_paul` 3.

**`ladderStep` у JSON:** після мапінгу `gameTopic` усі 205 питань сортуються за внутрішнім ключем «жорсткості» (на основі тієї ж евристики, що раніше давала 1–3, плюс довжини тексту для розв’язання нічий); потім **рівномірно** діляться на 10 сходинок (у поточному банку: кроки **1–5** по **21** питання, **6–10** по **20**). У `extraQuestions.ts` — поле **`ladderStep`** (для `eN`: `((N-1) % 10) + 1`).

**Фільтр колоди:** `buildDeck(topicFilter)` — для кожного раунду `step` 1…10 береться випадкове невикористане питання з `ladderStep === step` і обраною темою (або змішано); якщо пул порожній — послаблення до сусідніх сходинок, потім будь-яке з теми, у крайньому випадку дублікат з суфіксом `__dN`. Порядок у колоді — **за зростанням сходинки** (не перемішується весь список питань).

**Сесія (`useGame.ts`):**

| Поле / концепція | Призначення |
|------------------|-------------|
| `phase` | `'menu' \| 'playing' \| 'result'` |
| `sessionTimeLimit`, `timeLeft`, `timedOut` | ліміт секунд (`TIMER_SECONDS_PER_QUESTION`), зворотний відлік, прапорець програшу від часу |
| `pendingTopic` | тема до старту / під час гри для підпису в UI; **не** скидається під час гри |
| `deck` | масив `BibleQuestion` після `buildDeck`; індексація раундів тільки по `deck` |
| `roundIndex` | 0 … `deck.length - 1` |
| `current` | результат `shuffleAnswers` для активного питання |
| `activeQuestion` | елемент `deck[roundIndex]` (сирий) |
| `fiftyUsed`, `hiddenBy50` | **50:50 один раз за сесію** (як Запитати ШІ і вірш); `hiddenBy50` скидається в `loadRound` для нового питання, `fiftyUsed` — ні |
| `hallUsed` / `verseUsed` | один раз за сесію; **не** скидаються в `loadRound` |
| `hallPercents` | скидається в `loadRound` |
| `chooseTopic` | обрана тема + одразу старт гри (`buildDeck`) |
| `returnToMainMenu` | після «Грати знову»: `menu`, очищення `deck` і pending |
| `loadRound()` | бере `deck[roundIndex]`, shuffle відповідей, скид `hiddenBy50` і панелі залу (не скидає `fiftyUsed`) |

**Перемішування:** порядок питань у колоді — **фіксований** 1…10 за `ladderStep`; відповіді — `shuffleAnswers`.

## 10. Зафіксовані продуктові рішення

- **Теми:** змішано + **одинадцять** тематичних доріжок за `QuestionTopic` (див. `TOPIC_MENU` у `topics.ts`); відповідність тегам джерела — розділ 5.
- **Сходинки складності в банку:** **`ladderStep` 1…10** (рівномірно по банку Огієнка); **завжди 10 питань** за гру в порядку легше → важче + один таймер на питання.
- **Драбина:** динамічна під `deck.length` (`prizeLadderForCount`).
- **Безпечні сходинки:** немає. **Таймер на питання:** є (`TIMER_SECONDS_PER_QUESTION`); глобального таймера на всю гру немає.
- **Усі три підказки (50:50, Запитати ШІ, вірш):** по **одному разу за сесію** гри.

## 11. Деталі логіки підказок (для AI — не розходитися з кодом)

### 11.1. Запитати ШІ (`applyHall`, `audienceShares`)

- Генеруються **цілі відсотки** для A–D, сума **100**.
- Серед **доступних** варіантів (не прихованих 50:50) **правильний завжди має найбільший %**.
- Якщо після **50:50** на **тому ж** питанні вже був показаний зал (`hallPercents !== null`), виклик `applyFiftyFifty` **перераховує** опит (`refreshHall`). Якщо зал ще не викликали — після 50:50 зал не з’являється сам.
- Після переходу на **наступне** питання `hallPercents` стає `null` (панель залу ховається), навіть якщо зал уже «вичерпано» глобально — кнопка залу лишається неактивною.

### 11.2. Вірш (`applyVerse`, `closeVerseModal`)

- Один виклик за гру; відкриває модальне вікно.
- Якщо в активному питанні є `verseText` — показує `verseLocation` (якщо є) + `verseText`.
- Якщо `verseText` відсутній — показує: «Упс... до даного питання вірш відсутній».

### 11.3. 50:50

- Доступна **лише один раз за гру** (`fiftyUsed` не скидається в `loadRound`; скидається лише в `beginQuiz` та `returnToMainMenu`).
- Залишає правильний + один випадковий неправильний; приховані кнопки з класом `hidden`. На наступних питаннях усі чотири варіанти знову видимі, але кнопка 50:50 лишається неактивною.

### 11.4. UI-копірайт

- Під кнопками: нагадування, що **50:50, зал і дзвінок** — **по одному разу за гру**.
- Під кнопками: нагадування, що **50:50, Запитати ШІ і вірш** — **по одному разу за гру**.

### 11.5. Таймер

- Значення з `TIMER_SECONDS_PER_QUESTION` (зараз **120** секунд на питання).
- Інтервал `setInterval` 1 с у `useGame`; при `timeLeft <= 0` — `onQuestionTimedOut()` → `result`, `timedOut=true`.
- Очищення інтервалу: `clearQuestionTimer` у `loadRound`, `pickAnswer`, `beginQuiz`, `returnToMainMenu`, `onScopeDispose`.

## 13. Джерела питань (ідеї для імпорту та розширення)

Готового **єдиного** відкритого API «біблійський квіз українською з відповідями» зазвичай **немає**; зручно **комбінувати**:

1. **[RascalTwo/BibleTrivia](https://github.com/RascalTwo/BibleTrivia)** — застосунок-квіз; структура даних може надихнути; тексти часто англійською.
2. **[swvincent/BibleInJson](https://github.com/swvincent/BibleInJson)** / **[openbible](https://github.com/churchstudio-org/openbible)** — повні тексти Біблії в JSON для **генерації** питань (вірш, контекст), не готові MCQ.
3. **[BradyStephenson/bible-data](https://github.com/BradyStephenson/bible-data)** — персонажі, місця, зв’язки (CSV/структуровані дані) — добре для **тематичних** питань (міста, події).
4. **[josephilipraja/bible-cross-reference-json](https://github.com/josephilipraja/bible-cross-reference-json)** — перехресні посилання KJV; корисно для складних «де написано…» після перекладу/локалізації.

**Практична стратегія:** тримати власний банк (`questions.ts` + `extraQuestions.ts` або JSON), позначати `topic` / `ladderStep`; за потреби напівавтоматично генерувати чернетки з JSON-тексту + ручна перевірка. **Одного «ідеального» готового датасету українською з ліцензією на весь банк зазвичай немає** — реалістично комбінувати власні формулювання з ідеями з відкритих даних (див. посилання вище) і перевіряти факти за обраним перекладом.
